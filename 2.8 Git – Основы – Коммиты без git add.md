

1. **`git commit -a`**
    
    - Добавляет _все изменения отслеживаемых_ файлов и сразу коммитит.
        
    - **Не захватывает** новые untracked файлы.
        
2. **`git commit -am "<msg>"`**
    
    - То же + сообщение в одном шаге. Удобно для мелких правок.
        
3. **`git commit -amend` без `add`**
    
    - Если изменения уже есть в индексе, `--amend` обновит снимок.
        
    - Для _мелкого_ исправления можно: `edit`, затем `git commit --amend -a`.
        
4. **`git add -u` и `git commit`**
    
    - `-u` («update») — помещает **изменения + удаления** отслеживаемых, без новых файлов.
        
5. **Когда полезно?**
    
    - Экстренная правка орфографии в коде.
        
    - Быстрые «housekeeping»-коммиты (`chore: bump version`).
        
6. **Риски**
    
    - Легко «утянуть» лишние правки. Всегда проверяйте:
        
        - `git diff` (рабочая ↔ индекс)
            
        - `git diff --cached` (**после** `commit -a` _до_ push нельзя)
